How the auto-login works?

Understanding how your data is protected

⚠️ Important Notes

This extension uses client-side encryption. Your credentials are never sent to any server. The encryption key is stored in your browser's secure storage and is only accessible to this extension.

If you want to look at the actual implementation, you can check the source code on GitHub.

Why use auto-login?

Cardmarket logs you out constantly and the page doesn't have the option to keep you logged in, this feature is meant to save you time by automatically logging you in when you visit the page. By encrypting your credentials and storing them securely in your browser, we ensure that your data is protected.

Secure Encryption

Your Cardmarket credentials are encrypted using AES-256 encryption before being stored in your browser's local storage.

Algorithm: AES-256-GCM
Key Length: 256 bits

Key Generation

The extension generates a unique encryption key for your browser. This key is derived from your browser's unique identifiers and stored securely in your browser's secure storage.

Key Source: Browser-specific entropy
Storage: Chrome's secure storage
Access: Restricted to extension context

Encryption Process

When you enter your credentials, they are encrypted in real-time:

1. Username & password are collected
2. AES-256-GCM encryption is applied
3. IV (Initialization Vector) is generated randomly
4. Encrypted data is stored in chrome.storage.local
5. Only the extension can decrypt the data

Decryption Process

When you visit the login page, the extension automatically decrypts your credentials:

1. Extension checks if auto-login is enabled
2. Retrieves encrypted credentials from storage
3. Decrypts using the stored key
4. Fills in the login form automatically
5. Submits the form securely

Security Features

  • Browser-Local Storage: Credentials never leave your device
  • Unique Keys: Each browser gets its own encryption key
  • Automatic Cleanup: Credentials are cleared when you clear browser data
  • No Server Storage: All data stays in your browser's local storage
